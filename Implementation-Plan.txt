STRICT DIRECTIVES:
1. Do not over-engineer the state management. Do not use Redux/Zustand. Use native localStorage combined with React Router's state/navigation or simple useEffect hooks.
2. Prioritize data accuracy and UI simplicity.
3. Use highly communicative, non-technical Indonesian language in the UI.
PHASE 1: INITIALIZATION & SETUP
Execute the following environment setup precisely:
1. Initialize project: npm create vite@latest perawatan-motor-app -- --template react
2. Install UI dependencies: npm install -D tailwindcss postcss autoprefixer
3. Install Routing dependency: npm install react-router-dom
4. Initialize Tailwind: npx tailwindcss init -p
5. Configure tailwind.config.js to scan ./index.html and ./src//*.{js,ts,jsx,tsx}.
6. Inject Tailwind directives into src/index.css.

3.2 Routing & LocalStorage Logic:
1. Home.jsx (Input Phase):
   * Capture user input (Motor Type, Odometer).
   * On submit: Save the input to localStorage (e.g., localStorage.setItem('motorCareData', JSON.stringify({type, km}))).
   * Programmatically navigate to /detail using useNavigate().
2. MotorDetail.jsx (Result Phase):
   * On mount (useEffect), read localStorage.getItem('motorCareData').
   * Security/UX Check: If no data exists in localStorage, redirect user back to / (Home) immediately.
   * If data exists, run the filter logic against serviceData.json.
   * CRITICAL FALLBACK: If input_km exceeds the highest range_km_max in JSON, return a default fallback object for major checkups instead of crashing.
   * Pass data to ResultCard.jsx.
PHASE 4: UX/UI STRICT RULES
1. Odometer Illiteracy Validation:
   * Implementation: In Home.jsx, if km_input > 200000, block submission and show an alert: "Wah, angkanya terlalu besar. Coba cek lagi speedometer-nya, masukkan angka yang paling besar saja di depan koma." Do NOT save to localStorage if invalid.
2. Visual/Iconographic Translation:
   * Implementation: In ResultCard.jsx, map text to generic inline SVG icons (e.g., oil drop, gear, fan). Do not use external image files.
3. Data Reset Flow:
   * Implementation: In MotorDetail.jsx, provide an "Ubah Data Motor / Cek Motor Lain" button. When clicked, this button MUST clear the specific localStorage item and navigate back to / (Home).
